# Multi-agent system for analyzing IT career market
## 0. Тип используемой модели:
### 0.1. google/gemma-4-E2B-it с квантированием 4-bit
### 0.2. Запуск программы:
в терменале введите: python main.py --role "название профессии (на английском)"

## 1. Агенты (папка agents):
### 1.0. Файл agent.py:
Содержит родительский класс для всех остальных агентов. В нём реализованы инициализация и запуск агента (AutoModelForCausalLM -> Transformers -> Generator), а также очистка памяти от него
### 1.1. Основные агенты:
Наследуются от родительского класса Agent. В себе каждый класс содержит промт - собственное задание, а также классы наследуемые от **BaseModel** из **Pydantic**
### 1.2. Сохранение формата данных:
Передача корректных json файлов между классами обеспечивается контролём библиотеки **Outlines**, 
с помощью классов в агентах, которые формируют нужную структуру выходного json файла

## 2. Методы (папка methods):
### 2.0. Пути:
При помощи библиотеки **Pathlib**, независимо от системы, путь просчитывается от агента, который вызвал этот метод до нужной папки в которую ложится файл json или md, имя которого передал агент (такое же действие происходит и при чтении из файла)
### 2.1. Файл save_methods.py:
Содержит три функции: 

save_json - сохранение из сформированного словаря в нужный json; 

save_dict - сохраняет из файла json словарь; 

save_in_big_dict - переносит из малого словаря поля в общий
### 2.2. Файл save_report.py:
generate_markdown - функция создания отчёта

## 3. Примеры (папка examples):
### 3.0. Разделение данных: 
Мне показалось более уместным, чтобы файлы json лежали отдельно от файлов md
### 3.1. Папка jsons:
Данная папка хранит все полученные файлы json, причём только к финальному json файлу присоединяется имя профессии для отделения друг от друга
### 3.2. Папка reports:
Данная папка хранит все полученные файлы md, тоесть для каждой новой профессии создаётся новый report.md а для упомянутой используется перезапись
